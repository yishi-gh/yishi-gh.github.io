计算机系统漫游

可执行文件的生成

在 GCC + Linux 平台中，一个 C 源文件通常经历预处理、编译、汇编、链接四步，才变成可以运行的可执行文件。运行如下命令：

flowchart LR
    A[hello.c
C 源程序] -->|预处理 cpp| B[hello.i
展开头文件和宏]
    B -->|编译 cc1| C[hello.s
汇编程序]
    C -->|汇编 as| D[hello.o
可重定位目标文件]
    E[printf.o / libc.a / 其他 .o] --> F
    D -->|链接 ld| F[hello
可执行目标文件]
  • 预处理处理 #include、#define 和条件编译，得到 .i 文件。
  • 编译把 C 代码翻译成汇编代码，得到 .s 文件。
  • 汇编把汇编代码翻译成机器码，得到 .o 可重定位目标文件。
  • 链接把多个 .o 和库文件合并，完成符号解析与重定位，得到可执行文件。

这里的可执行文件，在 Windows 后缀名为 .exe，Linux 中留空或 .out（早期）。

注释、类型名、局部变量名、参数名在“编译”这一步就彻底丢失了。但“函数名”和“全局变量名”在编译成 .o 时还作为符号名存在，直到“链接”阶段如果没有选择 -g ，才会被最终丢弃。

计算机系统层次结构

计算机系统通过多层抽象让程序员不用直接面对所有硬件细节。ISA 抽象处理器指令，虚拟内存抽象主存和磁盘，文件抽象 I/O 设备，进程抽象正在运行的程序，虚拟机则进一步抽象整台计算机。

  • 文件提供了最基本的对 I/O 设备的抽象。根据 UNIX “一切皆文件”的哲学，所有资源甚至磁盘、进程、Socket 都被抽象为文件，可以通过统一的读写 API 访问。
  • 虚拟内存提供了程序存储器的抽象。在虚拟内存中，每一个程序都以为自己独占整个内存地址空间，拥有从0开始的连续地址空间。但实际上，这些地址可能分布在物理内存的各处甚至磁盘上。
  • 指令集架构 ISA 提供了对硬件的抽象，使得程序好像运行在一个一次只执行一条指令的处理器上。
  • 进程则提供对正在运行的程序的抽象。在 CPU 层面，系统通过时间片，使得 CPU 在多个任务间快速切换，形成了并行的假象。每个程序都以为自己持续运行，但实际上可能暂停了几百次。
  • 在最高层，虚拟机提供对整个计算机的抽象。这些能力由硬件和操作系统共同维护。

性能指标

CPU 时钟周期是处理器内部主时钟脉冲信号的宽度，可以看作 CPU 工作的最小时间单位。主频 表示每秒有多少个时钟周期。CPU 的性能由三件事共同决定：指令条数 (Instruction Count)、每条指令平均需要的周期数 (Cycles Per Instruction)、以及时钟频率 。

因此，主频高不一定程序更快。一个优化可能减少 ，也可能改变 ；一台机器主频更高，但如果每条指令平均更慢，最终执行时间仍然可能更长。

字节序与地址

多字节对象一般被存储为连续字节，对象地址通常是这些字节中最低的地址。Intel/x86-64 使用小端序。

  • 小端序把低位字节放在低地址，高位字节放在高地址；
  • 大端序相反。

x 的值和 x 的地址是两件事。&x = 0x500 只说明 x 从地址 0x500 开始存放，并不说明 x 的值等于 0x500。如果题目要求写成固定宽度，地址和数值都可以在高位补 0。例如地址 0x500 写成 32 位是 0x00000500，数值 1000 写成 32 位是 0x000003e8。

指令、寄存器与寻址

寻址方式是 CPU 根据指令信息定位操作数的方法。操作数可位于：指令中（立即寻址）、寄存器中（寄存器寻址）、存储单元中（存储器寻址，按字节编址）、端口中（端口寻址）。

x86-64 采用小端序存放多字节立即数，用 $ 标识。AT&T 语法通常写成 op 源操作数, 目的操作数，例如 sub S,D 表示 。

常用整数寄存器如下：

64 位 32 位 常见用途

%rbx %ebx 被调用者保存
%rcx %ecx 第 4 个整型参数、计数
%rdx %edx 第 3 个整型参数
%rsi %esi 第 2 个整型参数
%rdi %edi 第 1 个整型参数
%rsp %esp 栈指针
%rbp %ebp 基址指针/帧指针

%rip %eip 指令指针

其中，%rbx, %rbp, r12 ~ %r15 是被调用者保存，其余的是调用者保存。调用者保存解决的问题是：函数 A 调用函数 B 时，寄存器里的值怎么保证不被 B 弄丢？核心思想是：谁用谁保存，用完要恢复。每个通用寄存器要么由调用者保存，要么由被调用者保存。

调用者保存 被调用者保存
谁执行 调用者（A） 被调用者（B）
保存时机 调用前（call 之前） 被调用时（函数开头）
保存位置 调用者的栈帧 被调用者的栈帧
恢复时机 调用后（call 之后） 返回前（ret 之前）
目的 保护调用者自己的值 保护调用者的值不被自己破坏

换句话说，调用者保存就像，自己先备份再借给别人；被调用者保存就像，还车之前要洗车。他们保护的是同一个对象 A（即调用者）的所有参数，但保护的是参数的不同部分（即不同寄存器）。对于同一个寄存器，不会被二者同时保存。

寄存器访问速度远超内存，优先使用寄存器能显著提升性能。但在

  • 参数数量超过寄存器容量
  • 取地址操作必须为参数分配栈上空间
  • 参数是数组或结构体

时，会退而使用栈帧。

需要注意：整数的前六个参数和浮点数的前八个是各自独立排序的，即使整数排满了，第一个出现的浮点数仍然可以放到浮点数寄存器。

这些规则大多是 ABI 约定，不是 CPU 硬件强制。CPU 并不知道 %rdi 是第一个参数；这是编译器、库和操作系统共同遵守的二进制接口。相对地，%rip 和 %rsp 更接近硬件机制：CPU 靠 %rip 取下一条指令，push/pop/call/ret 会自动使用 %rsp。

寄存器保存也遵循约定。%rax、%rcx、%rdx、%rsi、%rdi、%r8-%r11 通常由调用者保存；%rbx、%rbp、%r12-%r15 通常由被调用者保存。

虽然调用顺序约定不是强制的，自己做实验时即使不遵循，机器也不会出错，但仍然不建议违反，这是协作开发的基础。

缓冲区溢出本质上是越界写破坏了栈帧中的其他内容。例如代码注入攻击，给程序一段可执行的攻击代码，剩下的部分一直填充到返回地址的起始，然后给出一个攻击地址。由于缓冲区溢出，攻击地址覆盖了原有的返回地址，ret 就可能跳到攻击地址执行攻击代码。常见防护包括栈随机化 ASLR、栈保护 canary、不可执行栈 NX Bit和避免危险输入函数。

GDB 与反汇编阅读

读反汇编时，可以先找参数寄存器，再找循环变量、累加变量和返回值。cmp S,D 要按 理解；jl/jg 等是有符号比较；call 会压入返回地址，ret 会弹出返回地址；sub $imm,%rsp 表示为当前栈帧分配空间。

修改 rsp 的指令主要包括直接修改和间接影响两类：

  • 直接修改通过 mov rsp, ...、lea rsp, [rbp - N]、add/sub rsp, N 以及 enter/leave 等指令显式改变栈指针；
  • 间接影响则是 push/pop、call/ret 等指令在压栈/出栈时由硬件自动更新 rsp，函数调用时 call 压入返回地址，ret 弹出返回地址，同时 push 减少 rsp，pop 增加 rsp。

链接

对应教材第 7 章。链接要解决的问题是：多个目标文件和库如何合成一个可以装入执行的程序。

目标文件主要有三类：可重定位目标文件 .o、可执行目标文件、共享目标文件 .so。可重定位文件还不能直接运行，它的代码和数据地址通常从 0 开始，等待链接器安排最终位置。可执行文件中的地址则是虚拟地址空间中的地址，可以被装入内存执行。

常见 ELF 节如下：

节 含义
.text 机器代码
.rodata 只读数据，如字符串常量、跳转表
.data 已初始化的全局变量和静态变量
.bss 未初始化的全局变量和静态变量，只占位，不占实际磁盘空间
.symtab 符号表
.rel.text / .rel.data 重定位信息
.strtab 字符串表

链接做两件核心工作。第一是符号解析，把每个符号引用绑定到确定的符号定义；第二是重定位，把多个代码节、数据节合并，并修正引用处的地址。

flowchart LR
    A[多个 .o 文件] --> B[符号表]
    B --> C[符号解析
引用绑定到定义]
    C --> D[重定位
合并节并修正地址]
    D --> E[可执行文件]

链接器符号分为三类：

符号类型 含义 例子
Global symbols 本模块定义，能被其他模块引用 非 static 函数、非 static 全局变量
External symbols 其他模块定义，本模块引用 extern void swap();
Local symbols 本模块定义且只在本模块使用 static 函数、static 全局变量

本模块的外部符号，在定义它的模块中就是全局符号。因此全局符号和外部符号是相对的。

这里的 Local symbols 不是函数里的普通局部变量，主要是使用 static 关键字修饰的函数、全局变量以及函数内的局部变量。函数内的其他自动局部变量和临时变量在编译后就消失了，通常位于栈或寄存器中，链接器并不关心它们的名字。

static 和 const 也要分开理解。static 主要影响存储期、链接属性或归属；const 主要表达“不能通过这个名字修改值”。例如文件作用域的 static int g; 是“只在本文件可见”的全局存储对象，而不是“更全局”的变量；函数内的 static int x; 仍然只能在函数内访问，但生命周期延长到整个程序运行期。

静态库 .a 是多个 .o 的归档文件。链接器从左到右扫描命令行，只取出能够解析当前未定义引用的库成员。所以库通常要放在使用它的目标文件之后；若两个库循环依赖，可能需要重复列出库，例如 libx.a liby.a libx.a。

gcc 中默认库 libc.a 无需显式指定。

Archiver（归档器）允许增量更新，只要重新编译需修改的源码并将其 .o 文件替换到静态库中。
