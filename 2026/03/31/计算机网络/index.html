开发导向的计网主要集中在传输层、应用层和少量网络层

计网架构

速记 OSI 七层架构：物联网叔会使用
物理层->数据链接层->网络层->运输层->会话层->表示层->应用层
TCP/IP 四层：
网路接口层->

下层为上层提供服务

osi的会话和表示太空洞，物理和数据链接层又太复杂。
tcpip的物理层和数据层分不清，且没有区分服务接口协议等概念，链路层也只是一个接口，但是已经被广泛使用
故常用五层架构：物联网
物理层->链路层->网络层->运输层->应用层

逻辑上各层之间都有互相通信的协议，但实际上是从上到下一直到物理层通信，再从下到上，走一个u字

物理层

OSI模型中，物理层是最底层，指的是物理信道，其作用是管理传输介质上的接口特性和规范，而不仅是传输介质本身

传输介质

常见有线传输介质：
- 双绞线（Twisted Pair）：两个铜线绞在一起，互相抵消波的影响。使用铝箔包裹的叫有屏蔽的双绞线（STP），没有的叫 UTP。入户一般是超5类的UTP，速率在1Gbps
- 光纤（Fiber）：光纤传输系统由光源，传输介质和检测器组成
- 同轴电缆（Co-axial）

传输标准命名规则：
1000 BASE - T X
传输速率 调制方式 传输媒介 编码体系

BASE ：baseband，基带信号，一根线缆只传输一个信号
BROAD：broadband，宽频信号，一根线缆可传输多个信号

T：双绞线
F：光纤
C：同轴电缆

空：曼彻斯特编码
x：快速以太网使用 4B/5B 为分组码
X：千兆以太网使用 8B/10B 为分组码
R： 使用 64B/66B 为分组码

编码

归零编码（RZ编码，Return-to-Zero Code）：正电平代表逻辑1，负电平代表逻辑0，输送完一位电平归零
不归零编码（NRZ）：输送完不归零
反向不归零编码（NRZI）：相比于前一电位，信号翻转表示0，不翻转表示1
曼彻斯特编码：升平表示1，降平表示0
差分曼彻斯特编码：相比于前一电位，有翻转为0，无翻转为1

数字调制

使用模拟信号表示比特（数字信息），将比特和信号之间的映射过程称为数字调制

码元是在固定时长里在信道上传输的电平状态，一个码元可以代表一或多个比特

波特率：单位时间传输的码元个数，单位波特 Baud，表示物理信号的变化，即通信的速率。而比特率则关注实际传输的数据量，即有用信息的速率

其中 B 为波特率， C 为比特率， N 为码元的离散状态数（每个码元所能表示的不同状态数量）

调平方法有：调幅（波动幅度），调频（频率），调相（波形平移）

正交振幅调制（QAM）：频率相同，结合调幅和调相

其中 M 是相位数， N 是振幅数

网络层

负责寻址和路由

  • 无连接服务
    每个数据包（数据报）都携带了足够的，独立的信息，不需要维护连接，每个数据包独立选择路径。但是缺点是不保证可靠性，数据包可能重复、丢失、乱序。常运用于能够接受丢包的场景，如视频通话或直播。

  • 有连接服务
    发送方和接收方确定一条虚拟的通信路径（虚电路）。虚电路在通信过程中始终存在。特点是建立连接、可靠传输、固定路径。适用于可靠性要求高的场景。

  • 路由
    指的是在网络层选择数据包从源到目的地的最佳路径的过程。路由涉及到路由表的维护和更新，通常由协议动态计算。由管理员手动修改路由表的，叫静态路由。

  • 转发
    在路由选择完成后，设备将数据包从一个接口转发到另一个接口的过程。

  • 距离向量路由算法（DV）
    > 告诉邻居我所知道的全网信息（整张路由表）

每个路由器维护一个路由表，路由表中有到达其他路由器的最短路径。通过定期与邻居交换路由表信息更新自己的路由信息。本质上是基于贝尔曼算法
1. 初始化：每个路由器只知道直连网络距离为1，其余都为不可达
2. 周期性广播：定期把自己的路由表发给所有邻居
3. 路由表更新
4. 收敛：若干轮交换后，每个路由器都知道到达所有目标的最短路径
缺点是容易出现路由环路问题（无穷计数），收敛慢，对拓扑变化敏感（不具备快速检测故障和重新收敛的能力）

  • 链路状态路由算法
    > 告诉全网我所知道的邻居信息（和邻居的链路状态）

每个路由器向全网广播自己与邻居的链路状态（如带宽、延迟），所有路由器都最终构建出相同的网络拓扑图。每个路由器用 D 算法计算出到各节点的最短路径即路由表
1. 邻居发现，找出谁是自己的邻居
2. 测量链路状态，路由器到达邻居的成本可以自动设置，也可以由运营商配置
3. 生成链路状态数据包
4. 泛洪数据包给全网，其中每一个新数据包发出都有序号递增，从而管理包的先后顺序
5. 本地运行 D 算法构建最短路径树，生成路由表

  • IP数据报
    20~60字节的首部+65535字节的数据部分
    首部构成：4bit版本+4bit首部长度+8bit服务类型+16bit数据报长度（字节）+16bit标识+3bit标志+13bit片偏移量+8bit生存时间+8bit上层协议+16bit首部校验和+32bit源IP地址+32bit目标IP地址+选项（可选，长度可变）
    除了数据报中最后一个分片，其他分片都必须是8字节的倍数

IP地址

32位二进制确定的网络中每台设备的唯一标识符。一个IP不是指向一个主机，而是一个网络接口。

  • 点分十进制表示法
    四个0~255的十进制数组成，转换成二进制是32个数，分为网络号（标识子网归属）和主机号（标识具体设备），以子网掩码分界
    将IP地址与子网掩码按位与，可以得到网络号

  • 特殊IP地址：
    0.0.0.0 本机地址，默认路由
    255.255.255.255 表示网络中的所有主机（路由器不转发）
    127.x.x.x 回环地址，用于本地软件的回环测试
    有一部分地址保留用于内部网络，称为私有地址。这部分地址不被公共转发，也不在公共互联网使用，

  • 子网划分
    将一个地址块划分为几部分，将一个大的网络分割成多个小的子网，供多个内部网络使用，但对于外部仍然像单个网络

  • CIDR 无分类域间路由选择
    CIDR使用一个斜杠后跟网络前缀长度（即网络部分的位数）来表示IP地址和子网的关系，这种方法不拘泥于以整个字节划分，而是可以落在32位中的任何位置
